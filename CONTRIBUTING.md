# Contributing to ClaudeCode Settings JSONs

このプロジェクトへの貢献を歓迎します！以下のガイドラインに従ってください。

## 貢献方法

### 1. 新しい設定パターンの追加

新しいユースケースに対応する設定を追加する場合：

1. 適切なディレクトリを選択または新規作成
   - `configs/basic/`, `configs/standard/`, `configs/advanced/` - セキュリティレベル別
   - `configs/mcp/` - MCP統合設定
   - `configs/skills/` - Skills関連
   - `configs/agent-team/` - Agent/Team設定
   - 新しいカテゴリが必要な場合は `configs/新カテゴリ名/` を作成

2. JSON設定ファイルを作成
   - 必須フィールド: `description`, `allowedTools`
   - 推奨フィールド: `disallowedTools`, `toolRestrictions`, `notes`
   - schema.jsonに準拠すること

3. 設定ファイルの例：
```json
{
  "description": "明確な説明",
  "allowedTools": ["tool1", "tool2"],
  "disallowedTools": ["tool3"],
  "toolRestrictions": {
    "tool1": {
      "maxFileSize": 1000000
    }
  },
  "notes": [
    "この設定の目的",
    "使用上の注意点"
  ]
}
```

### 2. 既存設定の改善

既存の設定を改善する場合：

1. 変更の理由を明確に
2. セキュリティへの影響を考慮
3. 後方互換性を維持（可能な限り）

### 3. ドキュメントの更新

設定を追加・変更した場合は、以下も更新してください：

- `README.md` - 新しい設定の説明を追加
- `EXAMPLES.md` - 使用例を追加
- 必要に応じてディレクトリ構成図を更新

## コーディング規約

### JSON フォーマット

- インデント: 2スペース
- UTF-8エンコーディング
- 末尾にカンマなし
- プロパティ名は英語（descriptionとnotesの値は日本語可）

### ツール名の規則

- ClaudeCodeの公式ツール名を使用
- MCPツールは `mcp__service__command` または `service-mcp-server-command` 形式
- カスタムツールは明確に文書化

### セキュリティガイドライン

1. **最小権限の原則**: デフォルトで必要最小限の権限のみ付与
2. **明示的な拒否**: 危険なツールは明示的に拒否
3. **制限の文書化**: toolRestrictionsで詳細な制限を設定
4. **コメント追加**: notesフィールドでセキュリティ考慮事項を説明

## プルリクエストガイドライン

### PRを作成する前に

- [ ] JSON構文が正しいことを確認（`jq`や`jsonlint`でチェック）
- [ ] schema.jsonに準拠していることを確認
- [ ] 既存の設定と重複していないか確認
- [ ] ドキュメントを更新

### PRの説明

以下を含めてください：

```markdown
## 変更内容
- 何を追加/変更したか

## 目的
- なぜこの変更が必要か

## 使用例
- どのように使うか

## セキュリティへの影響
- セキュリティリスクはあるか
- リスクがある場合、どう軽減するか

## テスト
- どのようにテストしたか
```

### レビュープロセス

1. 自動チェック（JSON検証）が通過
2. メンテナーによるレビュー
3. 必要に応じて修正
4. マージ

## 設定の分類基準

### Basic（基本設定）
- 読み取り専用
- ファイル変更不可
- コマンド実行不可
- 高セキュリティ環境向け

### Standard（標準設定）
- 一般的な開発に必要な権限
- 制限付きファイル編集
- 安全なコマンド実行
- ほとんどのプロジェクトに推奨

### Advanced（上級設定）
- ほぼ全ての権限
- 信頼できる環境のみ
- 経験豊富な開発者向け

## よくある質問

### Q: どの設定に追加すべきか分からない
A: 以下の順で検討：
1. 用途別（MCP、Skills、Agent）で分類可能か
2. セキュリティレベルで分類（Basic/Standard/Advanced）
3. 新しいカテゴリが必要か

### Q: ツール名の正式名称が分からない
A: Claude Code公式ドキュメントまたは実際のツール実行ログで確認

### Q: 設定をテストする方法は？
A: 
1. テストプロジェクトを作成
2. 設定を `.claude/settings.json` に配置
3. Claude Codeで実際に動作確認
