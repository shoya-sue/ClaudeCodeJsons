# GitHub Copilot CLI — Minimal 設定

## 役割と制約

このプロジェクトでは **読み取り専用モード** で GitHub Copilot CLI を使用します。
コードの理解・レビュー・探索に特化し、ファイルの変更は行いません。

## できること

- コードの構造・ロジック・依存関係の解説
- バグや問題点の指摘（修正は手動で行う）
- 正規表現・コマンドの生成・説明
- GitHub Issues / PR の読み取りと要約

## できないこと（このプロジェクトでは行わない）

- ファイルの作成・編集・削除
- Git コミット・プッシュ
- パッケージのインストール
- テストの実行

## プロンプトのコツ

1. **具体的に質問する** — 「このファイルは何をしているの？」より「`src/auth.ts` の `validateToken` 関数の動作を説明して」
2. **コンテキストを提供する** — `@ファイル名` で関連ファイルを参照
3. **一度に一つ** — 複雑な質問は分割する
4. **explore エージェントを使う** — `/agent:explore コードベースの認証ロジックを説明して`

## セキュリティ規則

- `.env` / シークレットファイルの内容を共有しない
- 本番環境の認証情報を含むファイルを参照しない
- `kubectl delete` / `terraform destroy` は実行しない
